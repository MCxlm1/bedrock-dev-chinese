存在多种文件类型用于_Minecraft_，全部用于导入内容。所有_Minecraft_文件都是ZIP压缩包，重命名为使用`mc…`扩展名。这些压缩包目前可分为三类：

-   **关卡（`mcworld`和`mcproject`）**：用于世界和项目的关卡数据及相关资源
-   **资源（`mcpack`和`mctemplate`）**：用于世界的装饰或辅助资源
-   **复合包（`mcaddon`和`mceditoraddon`）**：用于导入最多一个世界或项目以及任意数量的资源类型

Minecraft的所有文件类型都可以作为任何文件打开，启动Minecraft并导入内容。导入包时，它们会自动解压为其组成文件和目录。如果尚未打开，大多数文件类型将以普通模式启动Minecraft；`mcproject`和`mceditoraddon`将改为在编辑器模式下启动Minecraft。

## 关卡

关卡代表常规世界和编辑器项目的存档数据和资源。所有关卡，无论模式如何，都会被导入到`com.mojang`目录中的`minecraftWorlds`。

导入现有已保存关卡的完全重复副本将创建一个重复的已保存关卡。复合压缩包仅在包含多个关卡时导入其中一个，包括跨嵌套复合压缩包的关卡。

### 世界

`mcworld`
封装单个世界的压缩包

世界压缩包可以通过几种不同的方式创建：

-   压缩世界目录的_内容_并将扩展名从`zip`重命名为`mcworld`
-   在世界的"游戏设置"屏幕上使用"导出世界"按钮
-   在编辑器模式下，从文件→导出为→可玩世界菜单选项导出世界。世界将保存到`com.mojang`文件夹中的`projectbackups`目录。
-   在编辑器模式下，运行`/project export world`命令。世界将保存到`com.mojang`文件夹中的`projectbackups`目录。

在_Minecraft_以编辑器模式启动时导入世界包会将世界作为项目导入。导入的世界随后在编辑器模式之外将不可访问，需要重新导出为世界以供游玩。在世界压缩包中捆绑的编辑器扩展包在编辑器模式之外导入时将保留。

### 项目

`mcproject`
封装单个编辑器项目的压缩包

项目压缩包可以通过两种不同的方式创建：

-   压缩项目目录的_内容_并将扩展名从`zip`重命名为`mcproject`。
-   在世界的"游戏设置"屏幕上使用"导出项目"按钮
-   在编辑器模式下，运行`/project export project`命令。世界将保存到`com.mojang`文件夹中的`projectbackups`目录。

如果_Minecraft_未打开，启动`mcproject`文件将打开编辑器模式。如果_Minecraft_已打开但不在编辑器模式下，则导入`mcproject`将失败。

## 资源

资源压缩包表示多个非关卡内容的单一实例：

-   行为包
-   资源包
-   皮肤包
-   世界模板

所有资源压缩包都包含一个描述其内容的清单。如果资源压缩包的清单UUID和版本与同一类型的现有资源压缩包完全匹配，则会导入失败。请注意，行为包和资源包共享相同的UUID/版本空间。包含在世界、项目或模板中的自包含行为包和资源包在导入时不计算为重复项。

两种资源扩展名`mcpack`和`mctemplate`在功能上表现相同。最佳做法是为行为包、资源包和皮肤包使用`mcpack`，为世界模板使用`mctemplate`，以便更清楚地了解正在安装的内容。复合压缩包中可以包含任意数量的资源压缩包。

### 包

`mcpack`
代表单个行为包、资源包、皮肤包或世界模板的包。建议仅对世界模板使用`mctemplate`，而不是行为包、资源包或皮肤包。

包只能手动创建，通过压缩行为包、资源包或皮肤包目录的内容并将扩展名从`zip`重命名为`mcpack`。行为包和资源包是全局安装的，不会与安装在世界、项目或模板中的匹配行为包或资源包冲突。

#### 行为包

行为包附加到服务器以更改或扩展游戏玩法。行为包安装到`com.mojang`文件夹中的`behavior_packs`目录。

开发行为包必须手动放置在`com.mojang`下的`development_behavior_packs`目录中。

#### 资源包

资源包附加到客户端以影响声音、视觉效果等。资源包安装到`com.mojang`文件夹中的`resource_packs`目录。

开发资源包必须手动放置在`com.mojang`下的`development_resource_packs`目录中。

#### 皮肤包

皮肤包是仅限客户端的自定义皮肤包。皮肤包安装到`com.mojang`文件夹中的`skin_packs`目录。

开发皮肤包必须手动放置在`com.mojang`下的`development_skin_packs`目录中，但此功能似乎不起作用。

### 世界模板

`mctemplate`
代表单个行为包、资源包、皮肤包或世界模板的包。建议仅对世界模板使用`mctemplate`。

世界模板安装到`com.mojang`下的`world_templates`目录。世界模板可以通过几种不同的方式构建：

-   压缩世界目录的_内容_，添加世界模板清单，并将扩展名从`zip`重命名为`mctemplate`
-   在编辑器模式下，在世界的"游戏设置"屏幕上使用"导出模板"按钮
-   在编辑器模式下，运行`/project export template`命令。世界将保存到`com.mojang`文件夹中的`projectbackups`目录。

## 复合包

复合压缩包用于在单次导入操作中导入最多_一个_关卡压缩包以及任意数量或组合的资源压缩包。通常，复合包的内容必须被打包。也可以在复合压缩包的_顶层_提供目录，以便在不需要预先打包的情况下导入资产类型（行为包、资源包、皮肤包和世界模板）。组织用途的嵌套子目录不能使用。

复合内容按常规处理。例如，在编辑器模式下导入包含`mcworld`的`mcaddon`时，会将世界作为项目导入。

复合压缩包还可以包含任意数量或嵌套的其他复合压缩包，即使跨越_Minecraft_模式也是如此。嵌套复合压缩包不能用于绕过单个世界导入限制。

复合包只能通过压缩归档和资产类型手动构建。

### 附加包

`mcaddon`
通用复合内容压缩包

在_Minecraft_以编辑器模式启动时导入`mcaddon`包会将任何包含的世界作为项目导入。导入的世界随后在编辑器模式之外将不可访问，需要重新导出为世界以供游玩。资产类型按常规导入。

### 编辑器附加包

`mceditoraddon`
用于编辑器模式的复合内容压缩包

如果_Minecraft_未打开，启动`mcproject`文件将打开编辑器模式。如果_Minecraft_已打开但不在编辑器模式下，则导入`mcproject`将失败。
