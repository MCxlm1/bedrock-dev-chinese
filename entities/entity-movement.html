实体移动

实体移动
  • 移动速度
  • 移动类型
  • 移动修饰符

  • 导航能力
  • AI目标

  • 其他

在Minecraft中，实体有能力在世界中移动，无论是行走、游泳还是飞行。要获得这些行为，您的实体通常需要相当多的行为，分为不同类型。

在阅读本教程时，请记住您的实体至少需要：

  • 设置实体移动速度的组件。
  • 设置实体如何移动的组件（行走、飞行等）
  • 设置实体导航能力的组件，以便它可以生成路径。
  • 设置实体何时/何地移动的组件（AI目标）。

TIP

创建移动实体的最佳方法是从原版行为包中选择一个类似的实体，并将组件复制到您的实体中。

例如，Phantom、Ghast或Parrot都是飞行实体，但具有非常不同的游戏内行为！使用最接近的实体作为模板。

移动速度 ​

您的实体需要的第一件事是速度组件。这设置您的实体在世界中移动的速度。

组件注释
[minecraft:movement](file:///c/Users/MCxlm/Downloads/bedrock-wiki-wiki/bedrock-wiki-wiki/docs/entities/vanilla-usage-components#movement)设置移动速度（必需）
[minecraft:underwater_movement](file:///c/Users/MCxlm/Downloads/bedrock-wiki-wiki/bedrock-wiki-wiki/docs/entities/vanilla-usage-components#underwater-movement)设置在水中移动速度。
[minecraft:flying_speed](file:///c/Users/MCxlm/Downloads/bedrock-wiki-wiki/bedrock-wiki-wiki/docs/entities/vanilla-usage-components#flying-speed)设置空中速度。

您应该始终包含minecraft:movement。根据需要添加其他两个。

所有原版"游泳"实体如海豚都包含underwater_movement。只有一些飞行实体有flying_speed。目前还不知道为什么会这样。

移动类型 ​

您的实体还需要一个移动类型。移动类型为您的实体在世界中移动的_方式_设置硬编码行为。

您的实体只能包含一个移动类型。选择最接近您需求的组件。通常basic、amphibious和fly是不错的选择。

组件注释
minecraft:movement.amphibious此移动控制允许生物在水中游泳并在陆地上行走。
minecraft:movement.basic此组件增强实体的移动。
minecraft:movement.fly此移动控制导致生物飞行。
minecraft:movement.generic此移动控制允许生物飞行、游泳、攀爬等。
minecraft:movement.hover此移动控制导致生物悬停。
minecraft:movement.jump移动控制导致生物在移动时跳跃，跳跃之间有指定延迟。
minecraft:movement.skip此移动控制导致生物在移动时跳跃。
minecraft:movement.sway此移动控制导致生物左右摇摆，给人在游泳的印象。

移动修饰符 ​

移动修饰符提供有关您的实体如何在世界中移动的附加信息。这些组件对于普通实体不是必需的，但您应该了解它们。

组件注释
minecraft:water_movement设置实体在水中经历的摩擦力。
minecraft:rail_movement设置实体只能在铁轨上移动。
minecraft:friction_modifier设置实体在陆地上经历的摩擦力。

导航 ​

您的实体需要的下一件事是导航组件。导航组件有许多字段，比如实体是否能开门或避开阳光。如何设置这些字段通常比您选择的导航组件更重要！

有这么多导航组件的原因是，每个组件都提供略微不同的硬编码行为。选择名称/描述最符合您的实体将要执行的导航类型的导航组件。

在任何时候您只能有一个导航组件。

TIP

此组件非常重要。您应该查看原版示例，了解使用哪些字段和值的灵感。

组件注释
minecraft:navigation.climb允许此实体生成包括垂直墙壁的路径，就像原版蜘蛛一样。
minecraft:navigation.float允许此实体通过在空中飞行生成路径，就像常规的幽灵一样。
minecraft:navigation.generic允许此实体通过行走、游泳、飞行和攀爬以及上下跳跃方块来生成路径。
minecraft:navigation.fly允许此实体像原版鹦鹉一样在空中生成路径。
minecraft:navigation.swim允许此实体生成包括水的路径。
minecraft:navigation.walk允许此实体通过行走和像常规生物一样上下跳跃方块来生成路径。

导航能力 ​

除了移动和导航组件之外，还有很多附加组件来增强实体在世界中移动时的能力。

组件注释
minecraft:annotation.break_door允许实体破坏门。在导航组件中也必须启用。
minecraft:annotation.open_door允许实体开门。在导航组件中也必须启用。
minecraft:buoyant指定实体可以在哪些液体中漂浮。
minecraft:can_climb允许此实体爬上梯子。
minecraft:can_fly标记实体能够飞行。寻路器不会被限制在下方需要实心方块的路径上。
minecraft:can_power_jump允许实体像原版马一样强力跳跃。
minecraft:floats_in_liquid设置此实体可以在液体方块中漂浮。
minecraft:jump.dynamic定义动态类型的跳跃控制，将根据生物的速度修饰符改变跳跃属性。
minecraft:jump.static给予实体跳跃能力。

还有一些组件如minecraft:preferred_path，它将根据基于方块的路径成本修改导航。

AI目标 ​

导航组件告诉实体_如何_生成路径，但它没有说明_何时_或_何地_生成路径。这就是AI组件的作用。

AI目标以behavior为前缀，遵循优先级系统来选择运行哪个行为。较低的优先级将首先被选择。

通常，您应该添加相当多的AI组件，具有不同的优先级。这些叠加在一起，将为您的实体创建逼真的移动和行为。与往常一样，原版实体为添加哪些组件以及使用什么属性/优先级提供了良好的模板。

有太多生成路径的AI组件，无法在此文档中列出。以下是一些示例：

组件
minecraft:behavior.random_stroll
minecraft:behavior.follow_owner
minecraft:behavior.move_to_water
minecraft:behavior.stroll_towards_village

有关完整列表，请访问bedrock.dev。

寻路 ​

让实体前往特定地点是市场内容中最常见的请求之一。 最好的寻路方法是使用第二个实体，第一个实体会被吸引到这个实体。我将称这个次要实体为标记。如果您对如何创建标记感到困惑，请访问[虚拟实体](file:///c/Users/MCxlm/Downloads/bedrock-wiki-wiki/bedrock-wiki-wiki/docs/entities/dummy-entities)页面。

思路 ​

我们将要做的寻路实际上相当简单：让我们的实体对我们的标记具有攻击性，然后简单地将我们的标记放在我们希望实体前往的位置。难点在于知道要添加哪些组件以获得真正长距离的寻路。

组件 ​

可以根据需要编辑这些组件以创建良好的寻路。确保更新nearest_attackable_target以指向您的标记实体。这需要一个family_type，所以您应该在您的标记上设置一个。

不要忘记添加一些基本的移动和导航组件，以便您的实体能够移动。

json
"minecraft:behavior.nearest_attackable_target": {
    "priority": 0,
    "reselect_targets": true,
    "target_search_height": 1000,
    "within_radius": 1000,
    "must_see": false,
    "entity_types": [
        {
            "filters": [
                {
                    "test": "is_family",
                    "subject": "other",
                    "value": "waypoint_1"
                }
            ],
            "max_dist": 1000
        }
    ]
},
"minecraft:attack": {
    "damage": 0
},
"minecraft:behavior.melee_attack": {
    "priority": 0,
    "require_complete_path": true,
    "track_target": true
},
"minecraft:follow_range": {
    "value": 1000,
    "max": 1000
}

检测到达路径点 ​

您可以使用minecraft:target_nearby_sensor来检测何时到达标记实体：

json
"minecraft:target_nearby_sensor": {
    "inside_range": 2.0,
    "outside_range": 4.0,
    "must_see": true,
    "on_inside_range": {
        "event": "reached_waypoint"
    },
    "on_outside_range": {
        "event": "not_reached_waypoint"
    }
}
